## はじめに

私はコーディングエージェントを数値計算プログラミングに用いています。通常のソフトウェア開発と大体同じですが、数値計算特有の関心事がいくつかありますので、それを踏まえて私の工夫を紹介します。想定読者として数値計算の経験が大なり小なりあるが、コーディングエージェントを本格的に使ったことがない人を想定しています。また、基本方針はソフトウェアエンジニアリングの基本に則り、奇を衒ったことはやっていないつもりなので本職の方にはつまらないかもしれませんがご了承下さい。また、エージェンティックコーディングの大前提みたいな話も盛り込んでいますので慣れている方には「そこからか？」と思われるかもしれませんが、その点もご容赦願います。

あとでも出てきますが、先に私が使用しているツールを述べておくと、メインの開発はClaude Code (Max x5プラン)、メインレビュワーとしてCodex (Plusプラン)、サブレビュワーとしてCopilot (Proプラン)、CodeRabbit (Freeプラン、公開リポジトリのみ、Copilotの代わり)です。なぜ先にこの話をしているかというと、私のワークフローがこの構成に強く依存しているからです。したがって、同じプランを契約しないと全く同じワークフローを真似できない、という話になります。しかし、今回お伝えしたいのは、こういうツールを使うと良い、という話はもちろん含まれますが、どういう問題に遭遇したり、意識したりして現在のワークフロー構成になっているのか、ということです。私の開発ワークフローの進め方や知見はほとんどが開発用の自作エージェントスキルの[ultimatile/development-skills](https://github.com/ultimatile/development-skills)に集約されています。今回の記事はこれを解説する形になりますが、大事なのは「このスキルファイルをあなたも使ってください」という話**ではない**ことです。
なお、今回の進め方はいかに生成コードの信頼性を上げるか、という観点で組まれており、大量にトークンを消費します。ちょっとしたスクリプトや設定ファイルを書くなどの用途には過剰ですので注意して下さい。

## エージェントスキル

コーディングエージェント開発を進める上で最も大切なツールは[エージェントスキル](https://agentskills.io)だと思っています。以下単にスキルと書きます。直前にも私の手製開発用スキルが出てきました。この実体はYAMLフロントマター付きのマークダウンファイルで、所定の設定ファイル置き場に置くことで、エージェントが自動的に読み込んだり、明示的に読み込ませたりできるプロンプト用のファイルです。エージェントにやってもらいたいことを文章として渡して指示を出すわけですが、定型指示をこのファイルに記述してエージェントに渡すようにすることで、繰り返し似たようなプロンプトを書く必要がなくなり、再現性も高くなります。スキルはClaude Codeに限らず使用できるものとなっています。他人が作ったスキルを配布する機構も整備されつつあり、スキル用のパッケージマネージャーがあったりします(apmやgh skill)。ここで他人のスキルを使うかどうかの基準について述べたいと思います。エージェントに渡したい知識のようなWhatは他人のスキルでも良いが、どのように進めるかのHowはできるだけ自分で書いた方が良い、ということです。What/Howの区分けはそこまで明瞭ではないのですが、
