# はじめに

Rustでマルチプロセスを行う方法について解説します。

## プロセスとは

まずシステムの計算主体は主に2種類あり、1つはスレッド、もう1つはプロセスです。スレッドは同じメモリ空間を共有する軽量な計算単位であり、プロセスは独立したメモリ空間を持つ重い計算単位です。
マルチスレッディングに関しては、Rustの中心的関心であり、所有権と借用のシステムにより、スレッド間のデータ競合を防ぐことができます。しかし、プロセス間通信（IPC）については、Rustの標準ライブラリでは直接的なサポートはありません。

IPCを担うクレートとしてTokioが有名だと思います。Tokioはレイヤードクレートから構成される重厚なクレートです。あとで見るように、そもそも通信とは何か？何をしないといけないのかというのを整理する際に、引用することになります。
なお、先に断っておきますがTokioを使おうという話ではなく並列ランタイムのお手本として参照する形になります。

## message passaging interface (MPI)

ここで主にHPC分野で標準的に用いられるMPIについて軽く説明します。これは通信のハードウェア抽象化を担い、メッセージの送受信という形で統一されたインターフェースを提供します。抽象度としてちょうど中間に位置すると考えていますのでここで、取り扱います。
MPIが提供するAPIはpoint-to-point(p2p)通信とbroadcast通信(one-to-many, many-to-many)があります。これらはあるプロセスから他のプロセスにデータの送受信を行うというインターフェースで、ここではネットワークの詳細は隠蔽されています。つまり異なるプロセス間の通信であるという意味論しか持っておらず、同じ計算機内のプロセスの間の送受信なのか別の計算機との送受信なのか、というのをユーザーに区別させません。厳密には計算機のIDを調べることで特定は可能ですが、データの送受信にその区別は基本的に不要です。このようにハードウェア詳細は隠蔽しつつ、プロセスとプロセスとの間のやりとり、というOS上の意味論が剥き出しになっている、というのが特徴です。したがってここより下層では通信の詳細が剥き出しになり、逆に上層ではプロセス間の個別のやりとりが隠蔽される形になります。前者は大雑把にはネットワークライブラリ、後者を並列ランタイムと言います。
同様のレイヤのライブラリとしてNCCL、OpenSHMEM(NVSHMEM)などがあります。

## ネットワークライブラリ

ここでは通信の実体を担うライブラリを総称してネットワークライブラリと呼びます。最も有名なものはTCP/UDPでしょう。厳密に言えばこれはライブラリではなくプロトコルなためソケット通信ライブラリというのが正確かもしれません。同様のライブラリとしてUCX、libfabric、GASNetなどがあります。

## 並列ランタイム

並列ランタイムは通信を隠蔽し、ユーザーにアプリを書きやすくするためのライブラリ群です。
HPX、Legionなどがあります。

他の例としてはBLACSなどがありますが、これはScaLAPACKのバックエンド通信ライブラリで、このように分散並列アプリ専用のランタイムになっていることが多いです。
同様のアプリは、SLATE、dPLASMA、monarchなどがあります。

ここでPGASに触れておきます。
PGASとはpartitioned global address spaceの略で、通信モデルの一種です。
先ほど出てきたGASnetはその実装であり、PGAS言語群と呼ばれるPGASベースの分散並列言語がいくつか作られています。
RustではLamellarクレートがあります。
