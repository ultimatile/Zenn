# GLOSSARY.md（用語・表記の方針）

全用語を列挙する辞書ではない。**書き方が割れそうな語にだけ**集中して方針を固定する。
新しく迷う語が出たら、まず一般方針に当て、人名由来語は弁別軸で判断する。

**運用原則:** このガイドラインで判断できる語はそれに従う。判断できず **新造（新しい訳語・用語の造語）が必要な語** は、勝手に確定せず **具体案を添えてユーザーに確認**してから採用する。

## 英単語の基本方針

優先順位:

1. **日本語に訳す**（定着した訳語があればそれを使う）。
2. 適切な訳語・定着語がなければ**カタカナ**。
3. 訳すと意味がぶれる/損なう技術用語は**原語（英語）のまま**。例: `load-bearing`、（`thiserror` 文脈の）`reporter`。

ただし **人名は常にラテン文字フルスペル**。カタカナにしない（Gauss / Hermite / Jacobi / Gram / Hamilton）。
この綴りルールはカタカナ化より優先し、**定着語の中の人名部分にも適用する**（後述）。

語別メモ:

- **`public` / `private`**: コードのキーワード（`pub` 等）そのものを指すとき以外は**カタカナ**（パブリック／プライベート）。例: 「パブリックなエラー型」。`public API` のような定着英語の複合語は原語のまま。

## アンチパターン（避ける語）

表記の固定ではなく、**使わない語**を定める。新たに見つけたら同じ枠で追記する。

- **「AI」**: 原則禁止。指している対象を名指しする語（具体的なサービス名・製品名・手法名など）に置き換える。
  - 理由: 「知能（intelligence）」というものの定義自体が不確かで、「AI」が何を指すのか定まらない。
  - `LLM` への安易な置換も大体は不適切。多くの場合に指しているのは言語モデルそのものではない（その上に乗ったサービス・機能・エージェント等）ため、実体を名指しする。
  - **例外**: 社名・製品名など固有名詞の一部で使用が避けられない場合のみ可。

## 人名由来のテクニカルターム（最重要・最も割れる）

**綴り:** 人名部分は常にラテン文字。ガウス→Gauss、エルミート→Hermite、ヤコビ→Jacobi。

**語の作り方（弁別軸は「定着しているか／新造になるか」）:**

- 定着した概念は **定着語を使い、人名部分だけラテン綴り**にする。
  - エルミート行列 → **Hermite行列**
  - ガウス分布 → **Gauss分布**（訳語の「正規分布」を使うのも可）
  - ヤコビ行列 → **Jacobi行列**
- 定着していない概念に「人名＋名詞」を **新造しない**。一般方針/複合語で処理する。
  - 例: "Gaussian matrix" を「Gauss行列」とは書かない（その語は定着していない）。
  - `Gaussian〜` を禁じたのは**未定着の場合**の話。Gauss分布のような定着語は可。
- 対象そのものを指す独立した `-ian` 名詞で、頭名詞を持たないもの → **原語（-ian）のまま**。
  - **Hamiltonian**（演算子/関数そのもの。「Hamilton行列」等にしない）。
- 行列と行列式が同じ `-ian` 語に同居して衝突するもの → 行列には「人名＋行列」を使い、裸の `-ian` を行列の呼称に使わない（`-ian` は本来その**行列式**）。原文で出たら方言として注釈。
  - **Jacobian**: 行列の意味では**禁止** → **Jacobi行列**。原文で行列の意味の "Jacobian" が出たら「本来は Jacobi 行列式の意」と方言注釈する。

## Gramian（文脈依存）

分野間で意味が衝突するため、文脈で切り替える。

- **線形代数**: 行列は **Gram行列**、行列式は **Gram行列式**。裸の "Gramian" は本来「行列式」を指す（Jacobian と同じ構図）ので、行列の呼称として出てきたら方言として注釈する。
- **制御理論**: **可制御/可観測Gramian** は行列そのものを指す定着名（A\*A の行列式ではない積分行列）なので矯正せず、ラテン綴りで許容する。
- 上記2文脈に当てはまらない新造ケースは、運用原則どおり提案してユーザーに確認する。

## 用字（漢字/かな・表外字の扱い）

人名でも英単語でもない和語・漢語の用字（どの字で書くか）。表外漢字は一律でなく、著者の拘りの有無で分岐する。機械的な統制は textlint（[`prh.yml`](./prh.yml)）が担い、ここには判断の根拠を残す。

**弁別軸:** 表外字でも **著者が伝統・意味で選ぶ term は漢字**、**無拘りの表記ゆれはかな**。

- **函数**（非標準・house style）。常用形は「関数」だが、数学の伝統表記として **函数** を採る。表外字「函」を含むが、意味・伝統を理由に**意図的に選ぶ**ため漢字を採用する。`関数 → 函数` に正規化する（`機関数`・`金融機関数`・`通関数量` 等の `〜関+数` 複合語は lookbehind で除外）。
- 逆に、拘りのない表外字は**かな**。例: 躓く → **つまずく**（「躓」は常用外で、特に選ぶ理由がない）。
- 形式名詞は**かな**、実質名詞は**漢字**（`期待どおり`／`仕組み`）。これは表外字の話とは別軸の用字原則で、prh が機械統制する。

### 技術外来語の最終長音

デフォルトは**長音つける派寄り**。ただし **prh では機械化しない**（判断レーン）。
理由: prh は部分文字列置換なので、素朴な規則が複合語を壊す（`ユーザ→ユーザー` は「ユーザビリティ」を、`メモリー→メモリ` は「メモリーカード」を、`サーバ→サーバー` は「サーバント」を壊す）。安全な共起を列挙し尽くせないため、`arabic-kanji-numbers` を無効化したのと同じく書き手判断に委ねる。

語ごとの方針:

- **定着した単一形**を持つ語はその形に従う: ユーザー・サーバー・ドライバー・フォーマッター・リンター（長）。
- **文脈・複合語で形が割れる語は据え置き＝状況判断**。既定は数学・技術の collocation で短: パラメータ（`弧長パラメータ` を `弧長パラメーター` にしない）・高/低レイヤ・メモリ・コンピュータ。
- **ベクトル**が canonical（数学で不変）。**ベクター**は「ベクター画像」等の限定文脈の方言で、ベクタとは別扱い（無理に統一しない）。
